# A股数据可利用性分析

## 已使用

- `daily/`：量价基础数据，已用于收益、动量、波动、成交活跃度等特征。
- `metric/`：换手率、估值、市值等指标，已进入模型。
- `moneyflow/`：资金流向，已构造净流入和大单/小单比例特征。
- `basic.csv`：股票池过滤、ST/北交所排除、行业和市场板块约束。

## 新检查的数据

### market/

包含：

- `000001.SH.csv` 上证指数
- `000300.SH.csv` 沪深300
- `399006.SZ.csv` 创业板指数

每个指数都有 2016-01-04 至 2026-04-28 的日频数据。已接入市场特征：

- 指数 1/3/5/10/20 日收益
- 20 日波动率
- 成交额相对 5 日均值

实验结果：单个 market-feature MLP 的 Rank IC 有提升，但与当前三种子 ensemble 混合后没有改善最终低换手 open-to-open 策略，因此暂不纳入主策略。市场数据更适合用于前一日市场状态开关。

### index_weight/

包含沪深300和创业板指的月度成分权重文件，共 248 个 CSV。可用于：

- `is_hs300` / `hs300_weight`
- `is_chinext_index` / `chinext_weight`
- 与指数成分股做 benchmark 或子股票池对照

当前最终 Top8 均不是 2026-04 沪深300或创业板指成分，因此该数据对当前候选清单没有直接影响。它更适合作为报告里的对照实验，而不是主策略增强。

### news/

新闻数据从 2019-01-01 至 2026-04-28，共 2671 个 CSV。最近交易日新闻数量通常为数百条，例如 2026-04-28 有 818 条。

2026-04-28 快讯中，用股票代码、简称、名称直接匹配，能命中约 410 只股票。但存在明显歧义：

- `机器人` 既是股票名，也可能是普通行业词。
- `太阳能` 既是股票名，也可能是泛指概念。
- 仅用股票名匹配可能产生误报。

当前最终 Top8 在 2026-04-28 新闻中没有直接公司新闻命中，因此新闻不会改变当前建仓清单。

建议定位：

- 不直接进主模型，避免文本歧义和过拟合。
- 用作每日候选股风险扫描：如果候选股出现退市、风险警示、处罚、减持、亏损等关键词，人工复核后可顺延到下一候选。

新增脚本：

```powershell
python scripts\scan_candidate_news.py --data-dir "..\A股数据" --candidates outputs\rebalance_plan_final_n8_no_star_industry1.csv --output outputs\candidate_news_scan.csv
```

## 当前结论

最有价值的数据增强排序：

1. `market/`：已接入特征，但暂不进主策略；可用于市场状态开关。
2. `news/`：适合每日风险扫描，不建议直接训练。
3. `index_weight/`：适合对照实验和报告分析，对当前 Top8 影响小。
